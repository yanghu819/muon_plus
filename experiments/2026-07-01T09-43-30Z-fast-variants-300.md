## Hypothesis

The 128-step screen said Lite diagonal is useful but has extreme norm ratios.
This run asks which top-level idea controls that problem without throwing away
the useful signal: adaptive layer gating, trust gating, hard module masking,
low-rank covariance, or random sketch covariance.

## Mechanism

The best run is not just a lucky scale change. Its telemetry matches the
intended mechanism: `qkv/c_fc` keep alpha near `1.0`, while high-norm
`o/c_proj` directions are downweighted. Hard masking `o/c_proj` is worse,
which means those modules are useful but unreliable. The right move is
self-diagnosing strength control, not module deletion.

## Lessons

An implementation issue was caught mid-run: the first adaptive attempt only
entered blend logic on refresh/telemetry paths. Commit `b03a6791` fixed this by
making `NEWMUON_LAYER_ADAPTIVE` force per-step instrumentation and temporary
buffers, like Trust. The interrupted adaptive attempt is excluded.

GPU2 lease expired during the first sketch64 attempt; the environment was
reopened and only sketch64 was rerun from the same source SHA.

## Next Decision

Promote `NEWMUON_LITE_DIAG=1 NEWMUON_LAYER_ADAPTIVE=1` to a longer gate
before any full 6200-step run. The next useful test is whether the 300-step
advantage over Lite/Trust persists at 900-2100 steps.
